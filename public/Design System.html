ProfileHeader src/components/profile/ProfileHeader.tsx

Top of /profile. Stacks avatar → name → email → metadata row (.pfMeta). The metadata row shows, in order: role chip, country flag emoji (no chrome — just the glyph), and the Wackens chip. Wackens chip only renders when the user has at least one Wacken year saved; tapping the count reveals the year list via the native tooltip.

Wackens chip rules:

  • Reuses the shared Chip primitive with the default (muted gray) variant — same palette as a normal vira-lata role chip. Discreet on purpose: it's ambient metadata, not a CTA. No accent dot, no colored fill.
  • Label is plural-aware: {count} Wacken when count = 1, {count} Wackens otherwise. German collapses to {count}× Wacken for both cases. Strings live in ProfilePage_*.json under wackenCountSingular / wackenCountPlural.
  • Tooltip exposes the sorted list of years (wackenCountTooltip) — useful for inspecting veterans on the team.
  • Always sits to the right of the country flag and wraps on narrow widths via .pfMeta { flex-wrap: wrap }.
  • Replaces the earlier diamond-shaped .pfYearDiamond badge — kept the badge in pill form because users could not read the year count from a clipped diamond. An earlier red-accented version was reverted to the default gray Chip for visual quietness.

MoshSplitSection src/components/profile/MoshSplitSection.tsx

Collapsible profile section showing the current user's net balance from MoshSplit (split.viralatas.org) — the festival finance app used by the vira-latas at Wacken. Placed after <ConflictSection>, before <EditProfileForm>. Accepts only userEmail: string; fetches live data via POST /api/moshsplit/pitboss/v1/balances/external-summary (Vercel proxy → split.viralatas.org, bearer token from VITE_MOSHSPLIT_TOKEN).

Five render states:

  • loading — spinner chip in header, pulse-skeleton in body.
  • not_found — API 404; return null, component fully invisible.
  • error — non-OK response or network failure; orange ! chip in header; ⚠ Could not load MoshSplit data + CTA in body.
  • settled (balance = 0) — teal chip + "All settled 🤘" message + CTA.
  • active (balance ≠ 0) — red chip if owes, teal chip if owed + expense list + festival/total row + CTA.

Collapsible trigger layout: logo (22×22, border-radius 5px, remote SVG with wallet fallback) → label group (Oswald 14px + mono sub-label) → balance chip (right-aligned). Uses <Collapsible> from src/ui with defaultOpen={false}. Wrapper: margin: var(--s-3) var(--s-4), max-width 400px.

Chip palette: owes = rgba(--accent, 0.20) bg / --accent-hover text; owed = rgba(--signal-ok, 0.20) bg / teal text; settled = attenuated teal; error = rgba(255,165,0,0.20) bg / orange text. Font mono 11px tabular-nums.

CTA: full-width <a> element, "Open MoshSplit →", rgba(--accent, 0.08) bg, opens https://split.viralatas.org in new tab. Visible in settled, active, and error states.

Best Practices

  • Always use CSS variables for design tokens. Never hardcode colors, spacing, or typography values in components or CSS modules.
  • Keep inline styles minimal. Prefer CSS modules and utility classes for consistent styling.
  • Document variants and props. Every component should have clear documentation of its props, variants, and expected behavior.
  • Test contrast ratios. Ensure all text meets WCAG AA standards (4.5:1 for normal text).
  • Use semantic colors. Use --signal-* tokens for state indicators (live, warn, ok, lost) rather than arbitrary colors.
  • Scale consistently. All spacing, sizes, and font sizes should reference the defined scales.
  • Update this file with every change. The Design System is a living document—keep it in sync with the codebase.
