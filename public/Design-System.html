Design System · v2.2 · Wacken Open Air 2026

Viralatas
Metaleiros 🤘

A festival companion PWA for ~20 metalheads. Dark-first, offline-first. Feels like a backstage wristband, not a corporate app.

Visual relationship to Wacken Open Air: We reference the festival by using its 8 canonical stage colors as the only saturated palette in the app. We do not recreate, copy, or approximate Wacken's own graphic identity (logotype, mascot, stage graphics). Stage color assignments are fixed and must not be changed — they are the festival's identity, not ours to reinterpret.

Crew-state exception
Cards representing a vira-lata's physical location (camping, metal place, lost) may use a single low-opacity gradient + ambient glow to signal emotional state. The glow must be static — no animation. Total exception budget: 1 gradient + 1 text-shadow + 1 box-shadow, all at ≤ 50% opacity. This is the only context where gradients and glows are permitted; the masthead rule "no glow / no gradients" applies everywhere else.

Iconography

Single-stroke geometric line icons. 24 px frame, 2 px stroke, square caps, miter joins. No filled chunky icons (other than tab bar active states), no skeuomorphism, no genre clichés (skulls, flames, pentagrams) — the typography and stage palette already do the metal lifting. Lucide is the closest off-the-shelf match.

07

Motion

Motion is functional, not decorative. Three rules: feedback the tap, signal a state change, never block content. Default duration 160 ms for state, 240 ms for entrance, 320 ms for celebration (pick toggle). Reduce to 0 ms when prefers-reduced-motion.

color: muted → accent-hover

10

Empty & Error States

Offline-first apps hit empty states constantly — first launch, stale cache, quiet periods. Every list screen has a canonical empty pattern. Error states are separate: they signal a recoverable failure, not just absence of data.

Visual pattern — empty states
  • Centered layout · display:flex; flex-direction:column; align-items:center; padding:40px 24px
  • Single muted line of copy · --font-sans · 14px · --text-muted · max-width: 28ch
  • Optional DS icon (24px stroke, --text-faint) above the copy — not required
  • No illustrations. No "no data found" placeholders. Keep the metal voice.
  • These are transitions, not destinations — density should be low
Canonical empty states
/now — pre-festival

Nobody has picked a band yet.

/schedule — no results

No bands found. Try a different filter.

/my-picks — zero picks

No picks yet. First to save wins.

/announcements — empty mural

Canonical error states · src/components/ErrorState.tsx
  • Reusable component · props: variant: 'network' | 'sync' | 'auth' · onRetry?: () => void · message?: string
  • Each variant has a retry CTA button styled to DS (.btn class)
  • Retry button only rendered when onRetry prop is provided
  • Custom message overrides default copy when provided

Element inventory (Monument grammar)

  • 6 px full-height left rail (.locStrip) — colored by stageColor(band.stage) for bands; per-kind palette for metal_place / camping / lost. Replaces the old 4 px top strip — gives every card a quiet vertical anchor.
  • Body grid (.groupBody → .groupMain) — left column flexes (1fr) for content; right column auto-sizes (or 64 px when the duck is rendered).
  • Kicker + presence dot — mono · 10 px · uppercase · per-kind color. The .liveDot with glow is shown whenever the user is here, not just on band cards. Copy: "Live · you here" / "Special Zone — You Are Here" / "Base Camp" / "Off the grid".
  • Title — var(--font-display) · 26 px · 700 · uppercase · per-kind color. (Up from 22 px to anchor the card.)
  • Subtitle — mono · 11 px · uppercase · 0.04 em letter-spacing · muted variant of the kind palette.
  • Crew pills — sit directly under the subtitle inside the left column (no longer a separate row). Current-user pill keeps the accent-red treatment regardless of card kind.
  • Count tile (.groupCount) — soft transparent surface (rgba(255,255,255,0.02)) · 1 px rgba(255,255,255,0.08) border · display 22 px · "vira-latas" mono caption (8 px, no-wrap). Becomes a 64×64 square when the duck is rendered alongside; otherwise auto-width.
  • DuckButton (tile variant) — 64×64 rounded square · transparent background · matching soft border · 46 px duck image · cooldown drain uses square-cornered conic gradient. Renders only when isBandLive && isPicked && category !== 'ceremony'.
  • Footer action (.groupActions) — separated by a 1 px --border top divider. Hosts a ghost-style "I am weak 🤘" micro-link: --font-mono · 10 px · 600 · uppercase · 0.16em letter-spacing · --text-faint at rest · transparent background · no border. Hover lifts color to --text-muted, draws a --border-strong bottom rule, and eases letter-spacing to 0.18em. Press snaps to --accent-hover bottom rule. Visual decoupling makes it unambiguous that quack is the hero CTA and weak is a tertiary escape hatch — opposite emotional actions, opposite visual weight.
  • Card highlight — when youAreHere, the body bg shifts to --bg-elevated (or the kind's "you-here" gradient) and the rail itself is the accent color. No more separate border-left: 2px solid var(--accent) override.

Component ownership

  • DuckButton — injected into CrewGroupsSection via onDuck?: () => void · duckCooldownUntil?: number props · rendered inside the right twin-tile column (.groupCountCol) with inBody + tile props. BandCard on /schedule and /my-picks still uses the legacy 40 px circle (no tile prop).
  • "I am weak 🤘" — owned by CrewGroupsSection.tsx · rendered via onSkip prop from /now page · only when isUserHere && group.kind === 'band' && myPlan.status === 'current'.
  • Card kinds — band, metal_place, camping, lost. Same grammar; only the rail color, kicker color, count tile palette, pill palette, and gradient backdrop change per kind.
Visibility & offline summary
  • DuckButton shown when band.is_live === true AND user has a pick for the band AND band.category !== 'ceremony'
  • DuckToast shown when viralatas:duck-quack window event fires for any picked band the current user did not themselves quack
  • Cooldown 90 seconds · per user per band · localStorage · other users' cooldowns are fully independent
  • Offline press queued to offline_duck_quacks (IndexedDB) · optimistic cooldown shown immediately · flushed on reconnect by duck.flushOfflineDucks()
  • Web Push (background / closed app) VAPID push from send-duck-push Edge Function · title = band name · body = "🦆" · icon = /rubber-duck.png
  • Asset public/rubber-duck.png — 114 KB · metalhead vira-lata dog riding a yellow rubber duck · used at 30 px (button) and 36 px (toast)
