Crew-state exception
Cards representing a vira-lata's physical location (camping, metal place, lost) may use a single low-opacity gradient + ambient glow to signal emotional state. The glow must be static — no animation. Total exception budget: 1 gradient + 1 text-shadow + 1 box-shadow, all at ≤ 50% opacity. This is the only context where gradients and glows are permitted; the masthead rule "no glow / no gradients" applies everywhere else.

BandCard

Displays band information with pick toggles and attendance counts.

Variants: schedule, timeline, ranked, ceremony

Features: duck button, conflict indicators, stage badge, attendee avatars, corner weekday label (V1 ghost)

Day label (showDayLabel): optional corner ghost — mono 9px weekday abbr in --text-faint, top-right of the body column (.dayGhost). Only on schedule and ranked variants. Weekday comes from bandWeekdayKey() in bandTime.ts (CEST, same rules as bandDay()) via SchedulePage i18n keys. Lineup (/schedule): shown when no day tab is active (filters.day === null). Hidden when a single day filter is selected. Popular: always shown. My Wacken: not used — festival day comes from section headers (.dayHeader).

Attendance chip (attendanceChip, Phase 33): timeline variant on /my-picks ended rows only. Values attended | missed (not “Saw”). Mono pill .attendanceChipAttended / .attendanceChipMissed in .meta; hidePick on ended rows. Card background + time column = same as upcoming; status = 4px left stripe only (.stripeAttended → --signal-ok, .stripeMissed → --signal-warn). Stage color on .stageBadge only. Opacity: .cardAttended 72% · .cardMissed 62%. No conflict outline on ended rows. See My Wacken Page section for page-level layout.

Duck button visibility: only when band is currently live, the user has picked it, band.category !== 'ceremony', AND the global duck_enabled killswitch (Phase 21) is on. The killswitch is a single boolean on app_settings, fetched once at app boot via DuckEnabledProvider and toggled from Godlike Powers → Duck feature. When off, the duck button is not rendered anywhere; offline-queued ducks still flush on reconnect.

Pick toggle: StarIcon → PawIcon (canine + heel, −14°) in the 64px column at 28px; outline when unpicked, filled accent when picked. Bottom nav My Wacken tab stays 18px.

Popular rating mode (ratingStats, Phase 32): when sort mode is rating, ranked cards pass ratingStats instead of attendeeCluster. Footer strip (border-top): left column = count + optional You · {score} + optional missed skip; right column = paw + large avg (.ratingHero). Pick avatars hidden.

Metal Battle country flag: when band.genre === 'Metal Battle', the genre label (.genre) is prefixed with the competitor's country flag emoji — e.g. 🇭🇺 Metal Battle. The flag comes from getMetalBattleCountryFlag(band.slot_id) in src/services/metalBattle.ts (static slot_id→country map; ISO2→emoji, or a regional 🌍 for multi-country entries like Sub-Saharan Africa / Balkans). Slots with no confirmed representative return null and the label renders as plain Metal Battle. All other genres render unchanged. Country flags are an allowed emoji exception in-UI (see icon language note).

Offline-first apps hit empty states constantly — first launch, stale cache, quiet periods. Every list screen has a canonical empty pattern. Error states are separate: they signal a recoverable failure, not just absence of data.

Best Practices

  • Always use CSS variables for design tokens. Never hardcode colors, spacing, or typography values in components or CSS modules.
  • Keep inline styles minimal. Prefer CSS modules and utility classes for consistent styling.
  • Document variants and props. Every component should have clear documentation of its props, variants, and expected behavior.
  • Test contrast ratios. Ensure all text meets WCAG AA standards (4.5:1 for normal text).
  • Use semantic colors. Use --signal-* tokens for state indicators (live, warn, ok, lost) rather than arbitrary colors.
  • Scale consistently. All spacing, sizes, and font sizes should reference the defined scales.
  • Update public/vira-lata-ds.html with every change. The design system is a living document — keep it in sync with the codebase.
